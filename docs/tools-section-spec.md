# Спецификация раздела «Калькуляторы» (/calculators)

Контентный проект нового раздела сайта с онлайн-калькуляторами расхода материалов для ремонта.
Документ не описывает код — это техническое и SEO-задание для реализации.

Контекст сайта (для согласованности с существующей архитектурой):
- Next.js App Router, плоская структура роутов (`/`, `/work`, `/portfolio`, `/price`), без вложенных сегментов.
- Прайс-лист: `src/content/prices/data.ts` — 22 категории, ~630 позиций, поля `{ name, price, dim }`.
- Метаданные через `createMetadata({ title, description, url, noIndex, absolute })`, шаблон title в layout: `%s | Ремонт и отделка`.
- Нав: `src/components/Nav/Nav.tsx` и `Footer.tsx` — сейчас 4 пункта (Главная, Отделка, Портфолио, Прайс).
- Регистрация роутов вручную в `src/app/sitemap.ts`.
- Нет UI-кит библиотеки, нет react-hook-form/zod, нет мат. библиотек — калькуляторы придётся писать на нативном React state + ручной валидации.
- Бизнес локальный (Бердск/Академгородок, Новосибирск), но калькуляторы — универсальный инструмент, локализация не нужна в title/H1, только в перелинковке на услуги/прайс.

---

## 1. Название раздела и URL

**Рекомендация: `/calculators`, нав-пункт «Калькуляторы».**

Причина: реальные поисковые запросы формулируются как «калькулятор плитки», «расход шпаклевки калькулятор» — слово «калькулятор» ближе к спросу, чем «инструменты» (слишком общее, ассоциируется с физическим инструментом — молотком, дрелью). «Инструменты» как альтернатива возможна, но хуже соответствует intent.

Дочерние страницы — плоские, без категорийных URL (`/calculators/tile`, а не `/calculators/finishing/tile`) — по конвенции сайта и во избежание тонких категорийных страниц, конкурирующих за одни и те же запросы с хабом. Категории — только визуальная группировка на хабе, не отдельные роуты.

## 3. Общие требования ко всем калькуляторам (UX/архитектура)

- Клиентские компоненты (`'use client'`), пересчёт на каждое изменение поля — без кнопки «Рассчитать», без запросов к серверу.
- Единый визуальный паттерн: Поля ввода → блок результата (обновляется live) → пояснение → FAQ → похожие инструменты.
- Числовые поля: `type="number"`, `inputMode="decimal"`, шаг 0.01 для метров, валидация диапазона перед расчётом; при невалидном вводе — блок результата показывает подсказку вместо NaN/Infinity, а не расчёт с мусорными данными.
- Общий диапазон защиты от абсурдных значений: длина/ширина помещения 0.1–50 м, площадь 0.1–500 м², проценты запаса 0–30%, толщина слоя 0.1–50 мм.
- Единицы измерения подписаны у каждого поля явно (м, м², мм, %, шт.) — никогда не оставлять «голое» число.
- Округление количества штучных материалов (плитка, доски, упаковки) — всегда вверх (`Math.ceil`), т.к. частями материал не продаётся.
- Каждая страница — обязательный дисклеймер под результатом: *«Расчёт — количество материала по введённым данным, а не смета работ. Точный объём закупки уточняйте у продавца с учётом остатков на складе и раскроя»* (формулировка адаптируется под инструмент).
- Если норма расхода зависит от производителя (шпаклёвка, краска, стяжка, клей) — обязательное поле для ручного ввода фактического расхода с этикетки товара, с предзаполненным типовым значением по умолчанию и пометкой «типовое значение, уточните по этикетке вашего материала».
- Общие переиспользуемые части (спецификация, не код): `CalculatorField` (лейбл + input + unit + validation), `CalculatorResult` (карточка с главным числом + доп. метриками), `CalculatorFaq` (аккордеон), `RelatedCalculators` (карточки-ссылки).

## 4. Детальные спецификации инструментов ядра

### 4.1 Калькулятор плитки

**SEO**
- slug: `/calculators/tile`
- Title: `Калькулятор плитки — расчёт количества на пол и стены`
- Description: `Онлайн-расчёт количества плитки на комнату по размерам помещения и плитки, с учётом способа укладки и запаса. Расчёт упаковок.`
- H1: `Калькулятор плитки: сколько плитки нужно на комнату`

**Поисковый спрос:** «калькулятор плитки», «расчет плитки на пол калькулятор», «сколько нужно плитки на ванную», «расчет плитки на м2», «калькулятор плитки на стену». Пользователь либо уже купил плитку определённого формата и хочет понять хватит ли, либо планирует закупку и не знает сколько коробок брать.

**Структура страницы:** H1 → краткое пояснение (что считает, что не считает — не считает стоимость) → форма + live-результат → блок «как считали» (формула человеческим языком) → SEO-текст → FAQ → похожие инструменты → ссылка на прайс (облицовочные работы) и услугу «облицовочные работы».

**Поля ввода:**
| Поле | Тип | Ед. | Валидация |
|---|---|---|---|
| Способ ввода площади | переключатель: «Длина×ширина» / «Площадь напрямую» | — | — |
| Длина помещения | number | м | 0.1–50 |
| Ширина помещения | number | м | 0.1–50 |
| Площадь помещения (альт. режим) | number | м² | 0.1–500 |
| Площадь проёмов/зон без плитки (двери, мебель) | number, опционально, default 0 | м² | 0 – площадь помещения |
| Длина плитки | number, пресеты 300/600/1200 + custom | мм | 10–2000 |
| Ширина плитки | number, пресеты 300/600/300 + custom | мм | 10–2000 |
| Способ укладки | select: «прямая» / «диагональная» / «со смещением (в разбежку)» | — | влияет на дефолт запаса |
| Запас на подрезку | number, default по типу укладки (7% прямая, 15% диагональная, 10% со смещением) | % | 5–25, редактируемо |
| (расшир.) Плиток в упаковке | number, опционально | шт | 1–200 |

**Формулы:**
```
S_нетто = (длина × ширина) − проёмы   // либо S введена напрямую
S_плитки = (длина_мм × ширина_мм) / 1 000 000   // м²/шт
Кол-во_без_запаса = S_нетто / S_плитки
Кол-во_с_запасом = ceil(Кол-во_без_запаса × (1 + запас / 100))
Упаковок = ceil(Кол-во_с_запасом / плиток_в_упаковке)   // если указано
S_к_закупке = Кол-во_с_запасом × S_плитки
```

**Результат:** крупно — «Нужно ~N плиток» → под ним: площадь помещения, площадь с запасом, количество упаковок (если указан размер упаковки), выбранный % запаса. Пояснение: «Плитка продаётся упаковками фиксированного размера — остаток учтён округлением вверх».

**SEO-текст (тезисно, экспертный, без воды):** что считает калькулятор (количество штук/площадь плитки под конкретный формат и способ укладки), почему запас на диагональную укладку выше (больше подрезки по углам), что не учитывается автоматически (бой при транспортировке/укладке, обрезка у сложных участков — санузел с коммуникациями), как трактовать результат (это ориентир для закупки, не бюджет ремонта), частая ошибка — покупать «ровно в размер» без запаса и не хватать 1 упаковки на подрезку в углах.

**FAQ:**
- Какой запас плитки брать на пол, а какой на стены? — обычно совпадает с способом укладки, а не с полом/стеной; для сложной геометрии (ниши, короба) стоит увеличить вручную.
- Нужно ли вычитать площадь двери или окна? — да, если это цельный проём без плитки; мелкие элементы (розетки, углы) не вычитаются — они и формируют часть подрезки.
- Что делать с остатком плитки? — оставить про запас на ремонт/замену повреждённой плитки в будущем, у калькулятора остаток не идёт «в минус».
- Почему в магазине посчитали другое количество? — продавцы часто закладывают свой % запаса или округляют сразу к упаковкам без указания расчёта — сверяйте вводные (формат плитки, способ укладки).

**Перелинковка:** Расход плиточного клея, Расход затирки, Калькулятор наливного пола (стяжка перед укладкой) → прайс `/price#tiling` (Облицовочные работы) → услуга «Облицовочные работы» на `/work`.

### 4.2 Калькулятор обоев

**SEO**
- slug: `/calculators/wallpaper`
- Title: `Калькулятор обоев — расчёт количества рулонов`
- Description: `Расчёт количества рулонов обоев по периметру и высоте комнаты, ширине рулона и раппорту рисунка. Точный подсчёт полос.`
- H1: `Калькулятор обоев: сколько рулонов нужно на комнату`

**Поисковый спрос:** «калькулятор обоев», «расчет обоев на комнату», «сколько рулонов обоев нужно», «расчет обоев по площади комнаты», «раппорт обоев расчет».

**Структура:** аналогична плитке; отдельный акцент — блок «Почему считаем по периметру, а не по площади» (частый вопрос).

**Поля:**
| Поле | Тип | Ед. | Валидация |
|---|---|---|---|
| Периметр помещения (или Длина+Ширина для расчёта) | number / 2 числа | м | 0.1–100 |
| Высота потолков | number | м | 2.0–4.5 |
| Ширина рулона | select пресеты 0.53/0.7/1.0/1.06 м + custom | м | 0.3–1.5 |
| Длина рулона | number, default 10.05 | м | 1–25 |
| Раппорт рисунка (0 — без подгонки) | number | см | 0–100 |
| Количество дверных проёмов | number, default 0 | шт | 0–20 |
| Количество оконных проёмов | number, default 0 | шт | 0–20 |
| Запас на подгонку | number, авторасчёт по раппорту, редактируемо | % | 0–25 |

**Формулы:**
```
Высота_полосы = высота_потолков + 0.1 (запас на подрезку сверху/снизу)
// если раппорт > 0, высота_полосы округляется вверх до кратности раппорта:
Высота_полосы_с_раппортом = ceil(Высота_полосы / (раппорт/100)) × (раппорт/100)   // при раппорте > 0

Полос_из_рулона = floor(длина_рулона / Высота_полосы_с_раппортом)
Периметр_расчётный = периметр − (двери×0.8 + окна×0.6)   // усреднённая ширина проёма, применяется частично — двери и окна режут полосу, но не убирают её полностью
Полос_нужно = ceil(Периметр_расчётный / ширина_рулона)
Рулонов = ceil(Полос_нужно / Полос_из_рулона)
```

**Результат:** «Нужно ~N рулонов» → полос из рулона, всего полос, площадь стен, учтённый раппорт.

**SEO-текст:** почему расчёт идёт полосами по периметру, а не простым делением площади на площадь рулона (иначе не учитывается раскрой по высоте и раппорт); как раппорт увеличивает расход; что проёмы уменьшают расход не пропорционально площади (кусок над дверью часто идёт в отход).

**FAQ:**
- Как учитывать раппорт при расчёте обоев? — раппорт увеличивает высоту одной полосы за счёт подгонки рисунка, из рулона выходит меньше целых полос.
- Обои считаются по площади или по периметру? — по периметру и высоте, площадь используется только как проверочная величина.
- Нужно ли вычитать окна и двери? — частично, крупный проём сокращает нужное число полос, но не всегда пропорционально — над дверью обычно остаётся обрезок.
- Почему остаётся лишний рулон? — рулоны делятся на целые полосы, остаток по высоте идёт в отход, это нормально и заложено в расчёте упаковками.

**Перелинковка:** Расход обойного клея (фаза 3) → прайс `/price#wallpapering` (Поклейка обоев) → услуга на `/work`.

### 4.3 Калькулятор ламината

**SEO**
- slug: `/calculators/laminate`
- Title: `Калькулятор ламината — расчёт количества упаковок`
- Description: `Расчёт количества досок и упаковок ламината по площади комнаты, размеру доски и способу укладки, с учётом запаса на подрезку.`
- H1: `Калькулятор ламината: сколько упаковок нужно на комнату`

**Поисковый спрос:** «калькулятор ламината», «расчет ламината на комнату калькулятор», «сколько упаковок ламината нужно», «расчет ламината по площади».

**Поля:**
| Поле | Тип | Ед. | Валидация |
|---|---|---|---|
| Площадь / Длина×Ширина (переключатель) | — | м² / м | те же диапазоны, что у плитки |
| Длина доски | number, пресеты 1260/1285/1380 | мм | 300–2500 |
| Ширина доски | number, пресеты 190/192/193 | мм | 50–400 |
| Способ укладки | select: прямая / по диагонали | — | влияет на дефолт запаса |
| Запас | number, default 7% прямая / 15% диагональ | % | 5–20 |
| Досок в упаковке | number, опционально | шт | 1–20 |
| Площадь упаковки (альтернатива, если известна) | number, опционально | м² | 0.5–5 |

**Формулы:**
```
S_доски = (длина_мм × ширина_мм) / 1 000 000
Досок_без_запаса = S_помещения / S_доски
Досок_с_запасом = ceil(Досок_без_запаса × (1 + запас/100))
Упаковок = ceil(Досок_с_запасом / досок_в_упаковке)   // если указано
// либо, если известна площадь упаковки:
Упаковок = ceil(S_помещения × (1 + запас/100) / S_упаковки)
```

**Результат:** «Нужно ~N упаковок» (главное число, т.к. ламинат продаётся упаковками) → количество досок, площадь с запасом.

**SEO-текст:** отличие расчёта по площади помещения от расчёта «на глаз»; почему диагональная укладка требует большего запаса (больше косых резов по краям); подложка считается отдельно (площадь = площадь пола + 5%, упоминание, что это отдельный материал).

**FAQ:**
- Сколько нужно ламината с запасом на подрезку? — типово 7% при прямой укладке, 10–15% при диагональной, зависит от геометрии комнаты (ниши, выступы требуют больше).
- Как считать ламинат, если упаковки разного объёма у разных производителей? — вводите либо число досок в упаковке, либо площадь упаковки с этикетки — калькулятор посчитает под конкретный товар.
- Нужно ли отдельно считать подложку? — да, это отдельный материал, обычно берётся по площади пола + небольшой запас.
- Что делать с остатком досок? — сохранить на случай локальной замены повреждённой доски, т.к. один и тот же декор может со временем не переиздаваться.

**Перелинковка:** Калькулятор плинтуса (фаза 2), Калькулятор стяжки пола (основание под ламинат) → прайс `/price#flooring` (Напольные покрытия) → услуга на `/work`.

### 4.4 Расход шпаклёвки

**SEO**
- slug: `/calculators/putty-consumption`
- Title: `Расход шпаклёвки — калькулятор расчёта по площади`
- Description: `Расчёт расхода шпаклёвки в кг и мешках по площади стен, толщине слоя и типу материала. Учёт количества слоёв.`
- H1: `Калькулятор расхода шпаклёвки`

**Поисковый спрос:** «расход шпаклевки на м2», «калькулятор расхода шпаклевки», «сколько мешков шпаклевки нужно», «расход финишной шпаклевки на 1мм».

**Поля:**
| Поле | Тип | Ед. | Валидация |
|---|---|---|---|
| Площадь поверхности | number, либо расчёт через периметр×высоту−проёмы | м² | 0.1–500 |
| Толщина слоя | number | мм | 0.1–20 |
| Тип шпаклёвки | select: гипсовая финишная (default 0.9) / полимерная (1.0) / цементная базовая (1.4) / свой вариант | кг/м²/мм | — |
| Расход по этикетке (переопределение) | number, опционально, если заполнено — приоритет над select | кг/м²/мм | 0.3–3 |
| Количество слоёв | number, default 1 | шт | 1–5 |
| Запас | number, default 10% | % | 0–20 |
| Вес мешка | select 20/25/30 кг + custom, опционально | кг | 5–50 |

**Формулы:**
```
Расход_за_слой = площадь × толщина_мм × норма_кг_м2_мм
Итого_кг = Расход_за_слой × слоёв × (1 + запас/100)
Мешков = ceil(Итого_кг / вес_мешка)   // если указан вес мешка
```

**Важно (дисклеймер):** норма расхода жёстко зависит от конкретного производителя — калькулятор даёт типовые ориентиры по категории материала (гипсовая/полимерная/цементная), но точный расход всегда указан на упаковке купленного товара — рекомендуется ввести его в поле переопределения.

**Результат:** «~N кг шпаклёвки» → количество мешков, площадь, толщина, использованная норма расхода (явно показать какое значение применено — дефолтное или введённое пользователем).

**SEO-текст:** зачем нужен расход по слоям (стартовая/базовая и финишная шпаклёвка кладутся разной толщиной); почему цифра «может отличаться от факта» (качество основания, квалификация мастера, потери при замесе); как трактовать результат — это объём закупки материала, а не расход времени/денег на работу.

**FAQ:**
- Какой расход шпаклёвки на 1 мм слоя? — зависит от типа: гипсовые финишные обычно экономичнее цементных базовых, точное значение — на этикетке.
- Сколько мешков шпаклёвки нужно на комнату? — считается от площади, толщины и количества слоёв — введите свои размеры, калькулятор округлит до целых мешков.
- Почему в реальности ушло больше шпаклёвки, чем в расчёте? — неровности основания и потери при замесе/нанесении не входят в норму производителя, отсюда стандартный запас 10%.
- Нужно ли считать стартовый и финишный слой отдельно? — да, если это разные материалы с разным расходом — считайте калькулятором дважды.

**Перелинковка:** Калькулятор плитки (если поверхность потом облицовывается), Расход краски (шпаклёвка перед покраской) → прайс `/price#plastering`/`/price#painting` → услуга «Штукатурные/малярные работы».

**Поля:**

|---|---|---|---|
| Площадь окрашивания | number, либо через периметр×высоту−проёмы | м² | 0.1–500 |

```

```

**Результат:** «~N литров краски» → количество банок, площадь, слои, применённая норма.

**SEO-текст:** почему второй слой не удваивает расход линейно (частично впитывается в уже прокрашенную поверхность, но для расчёта закупки безопаснее считать по полной норме на каждый слой); влияние типа основания (шпаклёванная стена и голый бетон впитывают по-разному — оговорка, что расчёт ориентировочный); почему на банке указан диапазон, а не одно число (шероховатость и пористость основания).

**FAQ:**

- Как влияет цвет краски на расход? — тёмные и укрывистые пигменты обычно требуют меньше слоёв для перекрытия, светлые/яркие тона (красный, жёлтый) — больше, это стоит учесть количеством слоёв.

- Нужно ли считать грунтовку отдельно? — да, грунт — отдельный материал с собственной нормой расхода (см. соответствующий калькулятор, фаза 2).

### 4.6 Калькулятор стяжки пола

**SEO**
- slug: `/calculators/floor-screed`
- Title: `Калькулятор стяжки пола — расчёт смеси по площади`
- Description: `Расчёт количества сухой смеси и мешков для стяжки пола по площади и толщине слоя. Для готовых смесей и расчёта цемент/песок.`
- H1: `Калькулятор стяжки пола: расчёт смеси и мешков`

**Поисковый спрос:** «калькулятор стяжки пола», «расход стяжки на м2», «сколько мешков нужно на стяжку пола», «расчет цементно песчаной стяжки калькулятор».

**Поля:**
| Поле | Тип | Ед. | Валидация |
|---|---|---|---|
| Площадь помещения | number, либо длина×ширина | м² | 0.1–500 |
| Толщина стяжки | number | мм | 20–150 (предупреждение при <30 или >100 — нетиповые значения, часто требуют доп. армирования/маяков, уточнить у прораба) |
| Способ расчёта | select: «готовая сухая смесь» / «цемент+песок (объёмный расчёт)» | — | переключает набор полей ниже |
| — Расход смеси на м²/мм | number, default 1.8 кг/м²/мм (типовое значение для ЦПС смесей), редактируемо по этикетке | кг/м²/мм | 1.2–2.5 |
| — Пропорция цемент:песок (для объёмного расчёта) | select 1:3 (default) / 1:4 / custom | — | — |
| — Расход цемента на м³ смеси (для объёмного расчёта) | number, default 400 кг/м³ (типовое ориентировочное значение) | кг/м³ | 250–500 |
| Запас | number, default 10% | % | 0–20 |
| Вес мешка (для готовой смеси) | select 25/40 кг + custom | кг | 5–50 |

**Формулы (готовая смесь):**
```
Объём_л = площадь × толщина_мм   // л на м² при 1мм = 1л/м², итого площадь×толщина_мм = литры условного объёма
Итого_кг = площадь × толщина_мм × норма_кг_м2_мм × (1 + запас/100)
Мешков = ceil(Итого_кг / вес_мешка)
```

**Формулы (объёмный расчёт цемент/песок):**
```
Объём_м3 = площадь × (толщина_мм / 1000) × (1 + запас/100)
Цемент_кг = Объём_м3 × норма_цемента_кг_м3
Песок_м3 = Объём_м3 × (доля_песка_по_пропорции)   // напр. при 1:4 доля песка = 4/5 от объёма
```

**Результат:** для готовой смеси — «~N кг / N мешков»; для объёмного расчёта — «~N кг цемента, ~N м³ песка», плюс общий объём стяжки в м³.

**SEO-текст:** обязательная оговорка — расчёт смеси не заменяет проектный расчёт нагрузки на перекрытие и не учитывает армирование/маяки; разница между готовой сухой смесью (проще считать, стабильный расход) и объёмным расчётом (дешевле, но норма расхода цемента — усреднённый ориентир, зависит от марки цемента и влажности песка); почему тонкие стяжки (<30мм) обычно требуют других решений (наливной пол, фиброармирование).

**FAQ:**
- Какая минимальная толщина стяжки пола? — на практике меньше 30 мм цементно-песчаная стяжка плохо держит нагрузку без армирования, для тонких слоёв обычно применяют наливные смеси.
- Сколько мешков смеси нужно на стяжку 5 см? — зависит от площади и нормы расхода смеси конкретного производителя — введите значение с мешка для точного результата.
- Чем отличается расчёт «сухая смесь» от «цемент+песок»? — первый способ проще и точнее, если купили готовую фасованную смесь, второй — для самостоятельного замеса на объекте по пропорции.
- Нужно ли закладывать запас на стяжку? — да, из-за неровности основания и потерь при замесе, типовое значение 10%.

**Перелинковка:** Калькулятор наливного пола (фаза 2, для финишного выравнивания), Калькулятор ламината/плитки (финишное покрытие после стяжки) → прайс `/price#floor-screed` → услуга «Стяжка пола».

## 8. Рекомендации по реализации

**Запускать в первую очередь (наибольший SEO-потенциал + простая реализация):**
1. Калькулятор плитки — самый высокочастотный запрос в категории, простая формула, легко реализовать.
2. Калькулятор обоев — высокочастотный запрос, чуть сложнее (раппорт), но UX-паттерн переиспользуется от плитки.
3. Калькулятор ламината — высокочастотный запрос, формула почти идентична плитке (наименьшие доп. трудозатраты после плитки).

**Вторая волна (материалы, чуть более сложная логика допущений про бренд):**
4. Расход шпаклёвки, 5. Расход краски — почти идентичный паттерн друг с другом, можно разрабатывать параллельно, переиспользуя один и тот же компонент «расход по норме + переопределение».
6. Калькулятор стяжки пола — самый сложный из ядра (два режима расчёта), делать последним в первой волне, когда общий паттерн калькулятора уже обкатан.

**Добавлять позже (постепенное расширение):**
- Клей и затирка для плитки, грунтовка — прямое расширение уже реализованного паттерна «расход материала», низкая стоимость разработки.
- Плинтус — расширение паттерна «плитка/ламинат» (площадь → периметр).
- Наливной пол — по формуле похож на стяжку, но с другими нормами, стоит делать после стяжки, не одновременно (риск перепутать нормы расхода в контенте).
- Гипсокартон — самый сложный (два материала, схемы обрешётки), делать в последнюю очередь, когда раздел уже наберёт трафик и оправдает более сложную разработку.

**Общий принцип по приоритету:** сначала три калькулятора количества (плитка/обои/ламинат) — они дают наибольший объём брендонезависимого, объективно точного расчёта и максимальный трафик; затем калькуляторы расхода материалов, где корректность сильнее зависит от честного объяснения допущений (переопределение нормы расхода — обязательное поле, не опция).
